# 🌾 Feature Selection for CME Wheat Futures Forecasting

**Weekly forecasting of CME Wheat Futures (ZW) using market factors and systematic feature selection.**

> This project compares different feature selection methods + baseline for building a Lasso regression model on CME wheat futures data.

---

## 🎯 Project Goal

The main objective is to **exhaustively compare feature selection methods + baseline** and identify the best configuration for building a Lasso regression model on CME wheat futures data.

The pipeline covers:
- 🧪 Feature engineering (12 analytical categories)
- 🔄 Rolling-window cross-validation (2, 7, All Years training depths)
- 🧠 Multiple feature selection methods (filter, wrapper, metaheuristic)
- 📊 Performance evaluation with directional and error metrics
- 🎨 Publication-ready visualizations

---

## 📊 Data

Two datasets are used:

| Dataset | Description |
|---------|-------------|
| **Target** | `pr` — CME Wheat Futures (ZW) weekly price |
| **Factors** | Market factors grouped by type |

### Factor groups
| Prefix | Group | Examples |
|--------|-------|----------|
| `e` | Stock indexes | S&P 500, EuroNext 100, MSCI |
| `c` | Commodity indexes | Bloomberg Commodity Index, Brent Oil |
| `$` | Currencies | FX rates |
| `b` | Freight indexes | Baltic Capesize Index |
| `fp` | Futures positions | COT data (Prod_Merc, Swap, etc.) |
| `sd` | Supply & demand | Fundamental indicators |

The final design matrix has **~3,200+ features** over **704 weekly observations**.

## 🛠 Feature Engineering

Features are constructed **before** cross-validation to cover the whole dataset. Twelve analytical categories are generated by `generate_all_features()`:

| # | Category | What it captures |
|---|----------|------------------|
| 1 | 🧠 **Memory (Lags)** | `lag_1, 2, 3, 4, 12, 52` — inertia and annual seasonality |
| 2 | 🚀 **Momentum** | `diff_1, 4, 52`, acceleration, smoothed momentum |
| 3 | ⚖️ **Relative Strength** | `pr/e`, `pr/c`, `pr/$`, `pr/b` — price cleaned from market noise |
| 4 | 📊 **Channels & Levels** | Relative position, distance to SMA, SMA slope (12/52/104w) |
| 5 | 📉 **Volatility** | Spread, Z-score, annualized vol, vol % change |
| 6 | 🩺 **Trend Health** | Up-weeks count, Efficiency Ratio |
| 7 | 🔗 **Cross-Asset Correlations** | Rolling correlation (13w) between price and factors |
| 8 | 💥 **Distribution Tails** | Weeks since max, outlier flag (|Z| > 2.5) |
| 9 | 🌀 **Fractality** | Fractal dimension / Hurst proxy |
| 10 | 📆 **Seasonality** | Expanding mean by month (no leakage) |
| 11 | 🎚️ **Spread Dynamics** | Rate of convergence with benchmarks |
| 12 | 📏 **Stationarity** | Multi-period log returns (1w, 2w, 4w, 12w, 52w) |

Additional synergy features:
- **Volatility × Volume interaction** (futures positions × price z-scores)
- **Momentum spreads** (relative speed across asset classes)
- **Historical channel breakouts** (binary regime triggers)
- **Cumulative position flow** (COT flow integral)
- **Logistics cost pressure** (freight vs price)

## 🔄 Methodology

### Rolling-window validation
- **Start period:** week 264 (~5 years of history before first forecast)
- **Learning step:** 20 weeks
- **Training depths:** 2 years, 7 years, All available history
- **Horizon:** next-week log return → inverse-transformed to price levels

### Pipeline per window
```
Slice dataset by period
        ↓
Feature selection method
        ↓
StandardScaler (fit on train only — no leakage)
        ↓
LassoCV (TimeSeriesSplit, 150 alphas, eps=1e-15)
        ↓
Inverse-transform log-returns → price levels
        ↓
Compute metrics & store results
```

## 📈 Evaluation Metrics

For each rolling window and each method:

- **MAPE** — Mean Absolute Percentage Error on price levels
- **Directional Accuracy (DA)** — % of correctly predicted up/down moves
- **R² (train / test)** and **R² gap** — overfitting indicator
- **Alpha volatility** — stability of Lasso regularization path
- **Execution time** — speed comparison between methods

### Planned metrics
- # Selected Features
- Feature Stability Score (frequency of selection across windows)
- Jaccard Similarity between consecutive windows
